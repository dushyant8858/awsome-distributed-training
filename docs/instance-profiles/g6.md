> Covers: g6.xlarge through g6.48xlarge

## Key Characteristics

- **Ada Lovelace architecture** — supports FP8 Transformer Engine, 4th-gen
  Tensor Cores; nearly 2x the BF16 compute of A10G at similar VRAM
- **24 GB VRAM** — same capacity as g5 (A10G), but with FP8 support enabling
  better throughput for compatible workloads
- **Low memory bandwidth** (300 GB/s) — half of A10G's 600 GB/s; memory-bound
  workloads (large batch inference, attention) may see lower utilization
- **72W TDP per GPU** — low power, single-slot design; optimized for
  inference and light training
- **No NVLink, no GPUDirect RDMA** — same connectivity limitations as g5

## Distributed Training Considerations

- **Max model size**: Similar to g5 (~7B without offloading); FP8 can improve
  throughput but doesn't change memory capacity
- **FP8 advantage**: For models that support FP8 training (Transformer Engine),
  g6 offers ~4x the FP8 TFLOPS of g5's FP16 — significant for compute-bound
  workloads
- **Bandwidth limitation**: The 300 GB/s memory bandwidth (vs 600 GB/s on g5)
  can bottleneck memory-bound operations; measure before choosing over g5
- **Parallelism**: Same as g5 — FSDP2 with CPU offloading; TP over PCIe

## NCCL / EFA Configuration

> For complete NCCL/EFA tuning guidance, see the
> [EFA Cheatsheet](../../1.architectures/efa-cheatsheet.md).

```bash
# g6 does not support GPUDirect RDMA
export FI_EFA_USE_DEVICE_RDMA=0
export NCCL_PROTO=simple

# Standard EFA settings
export FI_PROVIDER=efa
export NCCL_SOCKET_IFNAME=^docker,lo,veth
```
