# p6e-gb200 Instance Family — NVIDIA GB200 Grace Blackwell

> Covers: p6e-gb200.36xlarge (single tray); 18 trays form an UltraServer

## Hardware at a Glance

| Spec | p6e-gb200.36xlarge (1 tray) | Full UltraServer (18 trays) |
|------|-----------------------------|-----------------------------|
| GPU | NVIDIA B200 (Blackwell) | NVIDIA B200 (Blackwell) |
| GPU count | 4 | 72 |
| VRAM per GPU | 185 GB HBM3e | 185 GB HBM3e |
| Total GPU memory | 740 GB | 13.4 TB |
| Memory bandwidth per GPU | 8,000 GB/s | 8,000 GB/s |
| BF16 TFLOPS per GPU | 2,250 (dense) / 4,500 (sparse) | 2,250 (dense) / 4,500 (sparse) |
| FP8 TFLOPS per GPU | 4,500 (dense) / 9,000 (sparse) | 4,500 (dense) / 9,000 (sparse) |

## Key Characteristics

- **72-GPU NVLink domain** — all 72 GPUs across 18 trays communicate via
  NVLink 5.0 at 1,800 GB/s per GPU, forming the largest coherent GPU
  interconnect on AWS; no EFA needed for intra-UltraServer communication
- **NVIDIA Grace CPU** — ARM-based (aarch64), not x86; software must be
  compiled for ARM. Uses chip-to-chip coherent link (not PCIe) to connect
  Grace CPU to Blackwell GPUs
- **13.4 TB total GPU memory** — enables training of the largest models
  (500B+ parameters) within a single NVLink domain without inter-node
  communication overhead
- **360 PFLOPS FP8** across the full UltraServer — designed for the largest
  foundation model training runs

## Distributed Training Considerations

- **Architecture is fundamentally different**: Grace (ARM) CPU, no PCIe
  between CPU and GPU (uses coherent chip-to-chip link), and the 72-GPU
  NVLink domain changes how parallelism strategies should be configured
- **Model parallelism within NVLink domain**: Tensor parallelism across up to
  72 GPUs without crossing an EFA boundary — vastly more efficient than
  8-GPU NVSwitch + EFA for inter-node TP
- **Software compatibility**: Requires ARM (aarch64) builds of all software
  including drivers, NCCL, and ML frameworks. Most NVIDIA container images
  provide ARM variants
- **Scheduling**: The UltraServer is scheduled as a unit; you get all 18 trays
  or none. Individual trays are not independently schedulable

## NCCL / EFA Configuration

> For complete NCCL/EFA tuning guidance, see the
> [EFA Cheatsheet](../../1.architectures/efa-cheatsheet.md).

```bash
# p6e-gb200 supports GPUDirect RDMA
export FI_EFA_USE_DEVICE_RDMA=1

# Standard EFA settings (for inter-UltraServer communication)
export FI_PROVIDER=efa
export NCCL_SOCKET_IFNAME=^docker,lo,veth
```

**Note:** Within a single UltraServer, all 72 GPUs communicate via NVLink
and do not use EFA. EFA is used for communication between UltraServers in
multi-UltraServer training setups.
