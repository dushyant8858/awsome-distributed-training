# g5 Instance Family — NVIDIA A10G

> Covers: g5.xlarge through g5.48xlarge

## Hardware at a Glance

| Spec | g5.12xlarge | g5.48xlarge |
|------|-------------|-------------|
| GPU | NVIDIA A10G (Ampere) | NVIDIA A10G (Ampere) |
| GPU count | 4 | 8 |
| VRAM per GPU | 24 GB GDDR6 | 24 GB GDDR6 |

## Key Characteristics

- **Cost-effective GPU instances** for training experimentation and fine-tuning
- **24 GB VRAM** — sufficient for models up to ~7B with BF16, or ~13-20B with
  FSDP2 + full CPU offloading
- **Ampere architecture** — supports BF16 and TF32 Tensor Cores, structured
  sparsity; no FP8
- **No NVLink** — GPU-to-GPU communication over PCIe Gen4; tensor parallelism
  is less efficient than on p-family instances
- **No GPUDirect RDMA** — EFA traffic goes through CPU, requiring
  `NCCL_PROTO=simple` and `FI_EFA_USE_DEVICE_RDMA=0`
- **Single EFA adapter** — inter-node bandwidth is significantly lower than
  p5 (1 vs 32 adapters); multi-node scaling is limited

## Distributed Training Considerations

- **Max model size**: ~7B without offloading (BF16); ~20B with FSDP2 + full
  CPU offloading on g5.12xlarge (4 GPUs, 192 GiB system RAM)
- **Parallelism**: FSDP2 with CPU offloading is the primary strategy; tensor
  parallelism over PCIe is possible but slow
- **Key lesson (veRL GRPO port)**: Porting from p5en required 11 OOM
  iterations. Critical changes: FSDP1→FSDP2, `offload_policy=True`,
  `model_dtype=bf16`, `enforce_eager=True`, `tensor_parallel_size=4`
- **Checkpoint sizing**: A 20B model produces ~117 GB checkpoints; with
  `save_freq=1` on 1.2 TB storage, disk fills in ~9 steps

## NCCL / EFA Configuration

> For complete NCCL/EFA tuning guidance, see the
> [EFA Cheatsheet](../../1.architectures/efa-cheatsheet.md).

```bash
# g5 does not support GPUDirect RDMA
export FI_EFA_USE_DEVICE_RDMA=0
export NCCL_PROTO=simple

# Standard EFA settings
export FI_PROVIDER=efa
export NCCL_SOCKET_IFNAME=^docker,lo,veth
```
