## Key Characteristics

- **Blackwell Ultra** — highest-performance GPU available on AWS; 288 GB HBM3e
  per GPU with 12-high HBM stacks
- **2.1 TB total GPU memory** per node — enables training of very large models
  (100B+) without pipeline parallelism or extensive offloading
- **6,400 Gbps network** — 2x the network bandwidth of p5en/p6-b200; 17 EFA
  adapters (EFAv4) for maximum multi-node scaling
- **4 TiB system memory** — 2x the system RAM of p5en/p6-b200; ample room
  for CPU offloading, data caching, and preprocessing
- **13.5 PFLOPS FP4** per GPU (dense) — Blackwell Ultra's FP4 capability enables
  next-generation quantized training workflows

## Distributed Training Considerations

- **Max model size**: 100B+ without offloading (BF16, TP=8); 288 GB per GPU
  makes most models fit without memory optimization tricks
- **Ideal for**: Largest-scale training runs — foundation models, MoE
  architectures, long-context models that need both compute and memory
- **Network advantage**: 6,400 Gbps (2x p6-b200) with 17 EFA adapters makes
  this the best choice for workloads that scale to many nodes
- **Cost consideration**: Most expensive instance type; use p6-b200 or p5en
  if the extra VRAM and network bandwidth aren't needed

## NCCL / EFA Configuration

> For complete NCCL/EFA tuning guidance, see the
> [EFA Cheatsheet](../../1.architectures/efa-cheatsheet.md).

```bash
# p6-b300 supports GPUDirect RDMA
export FI_EFA_USE_DEVICE_RDMA=1

# Standard EFA settings
export FI_PROVIDER=efa
export NCCL_SOCKET_IFNAME=^docker,lo,veth
```
