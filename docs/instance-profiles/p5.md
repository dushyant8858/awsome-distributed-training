> Covers: p5.48xlarge

| Spec | p5.48xlarge |
|------|-------------|
| GPU | NVIDIA H100 (Hopper, SXM5) |
| GPU count | 8 |
| VRAM per GPU | 80 GB HBM3 |
| Memory bandwidth per GPU | 3,350 GB/s |
| BF16 TFLOPS per GPU | 1,979 (dense) / 3,958 (sparse) |

| Local NVMe | 8 x 3.8 TB |

## Key Characteristics

- **H100 Hopper architecture** — 4th-gen Tensor Cores, FP8 Transformer Engine,
  ~6x the BF16 compute of A100
- **3.35 TB/s memory bandwidth** — critical for large-batch training and
  attention-heavy workloads
- **NVSwitch full-mesh** — 900 GB/s per GPU (50% more than p4d/p4de's 600 GB/s);
  tensor parallelism is highly efficient
- **32 EFA adapters** — 3,200 Gbps aggregate inter-node bandwidth; enables
  efficient multi-node scaling to hundreds of nodes
- **PCIe Gen4** (AMD EPYC) — note this is Gen4, not Gen5; CPU-GPU transfers
  are slower than p5en (which has PCIe Gen5)

## Distributed Training Considerations

- **Max model size**: ~30B without offloading (BF16, TP=8); 100B+ with PP +
  FSDP across multiple nodes
- **Primary training workhorse**: Most large-scale training runs on AWS use
  p5 or p5e/p5en instances
- **FP8 training**: H100's Transformer Engine enables FP8 training with
  automatic mixed precision — can nearly double throughput for compatible
  models
- **Parallelism**: Full flexibility — DP, FSDP, TP, PP, expert parallelism;
  NVSwitch + 32 EFA adapters support all strategies efficiently

## NCCL / EFA Configuration

> For complete NCCL/EFA tuning guidance, see the
> [EFA Cheatsheet](../../1.architectures/efa-cheatsheet.md).

```bash
# p5 supports GPUDirect RDMA
export FI_EFA_USE_DEVICE_RDMA=1

# Standard EFA settings
export FI_PROVIDER=efa
export NCCL_SOCKET_IFNAME=^docker,lo,veth
```
